Chapter 1 — Foundation

What is the Internet of Things?

The Internet of Things (IoT) is the network of physical objects — devices, vehicles, appliances, and environments — embedded with sensors, software, and connectivity that allows them to collect and exchange data over the internet. It bridges the gap between the physical world and digital systems, enabling unprecedented automation and intelligence.

THE IoT ECOSYSTEM — HOW IT ALL CONNECTS

What Makes IoT Unique?

IoT differs from regular internet by connecting things — physical objects with sensors — not just computers. Devices often have limited power (batteries), limited processing, intermittent connectivity, and must operate autonomously for months or years with no human interaction.

M2M vs IoT

M2M (Machine-to-Machine): older term, direct device-to-device communication, often closed networks (telecom-centric). IoT: broader ecosystem, IP-based, includes cloud platforms, analytics, AI, and open standards. IoT evolved from M2M by adding the internet backbone and application layer.

IoT vs IIoT vs AIoT

IoT: consumer + industrial devices connected to internet.
IIoT: Industrial IoT — factories, oil rigs, power grids.
AIoT: AI + IoT — intelligent edge inference (TinyML), predictive maintenance.
CIoT: Consumer IoT — smart home, wearables.

Why Layered Architecture?

Separation of concerns allows each layer to evolve independently. A new sensor type doesn't require changes to the cloud platform. A new analytics tool doesn't require touching the device firmware. This modularity enables the massive scale of IoT ecosystems.

Edge Computing in IoT

Processing data at the edge (on or near the device) reduces latency, saves bandwidth, improves privacy, and works offline. A factory robot can't wait 500ms for a cloud response — it needs <1ms local decisions. TinyML (TensorFlow Lite) enables ML at the edge.

Digital Twin

A digital twin is a virtual replica of a physical IoT device or system, updated in real-time via sensor data. Used for simulation, predictive maintenance, remote monitoring, and "what-if" analysis. Key concept in Industry 4.0 and smart cities.

Microservices + IoT

Modern IoT platforms use microservices architecture: device registry, message broker, data ingestion, stream processing, storage, and notification are separate services. This enables independent scaling — the message broker might handle 1M messages/second while analytics runs on demand.

MQTT Deep Dive

MQTT (Message Queuing Telemetry Transport) was created by IBM in 1999 for pipeline SCADA. Publish-subscribe model via a broker. Three QoS levels: 0=fire and forget, 1=at-least-once, 2=exactly-once. Retained messages give new subscribers the last value immediately.

CoAP — Constrained Application Protocol

IETF RFC 7252. Designed for very constrained devices (8-bit MCUs, lossy networks). Uses UDP instead of TCP (lower overhead). REST-like (GET/POST/PUT/DELETE) but binary-encoded. Supports multicast for device discovery. Maps to HTTP via proxy.

I2C Bus (Wire Library)

Two-wire serial: SDA (data) + SCL (clock). Multi-master, multi-slave. Each device has 7-bit address (up to 127 devices on one bus). Speeds: 100kHz (standard), 400kHz (fast), 3.4MHz (high-speed). Pull-up resistors required. Used by most sensors.

Chapter 6 — Connectivity

IoT Networks & Wireless Technologies

No single wireless technology fits all IoT scenarios. The choice depends on range, data rate, power consumption, and cost. From Bluetooth inside your ear to LoRa across a 15km farm — each technology fills a specific niche.

WIRELESS TECHNOLOGY RANGE vs DATA RATE MAP

LoRa / LoRaWAN

Long Range (LoRa) uses chirp spread spectrum modulation on unlicensed sub-GHz bands (868MHz EU, 915MHz US). Range: 2–15km rural, 2–5km urban. Battery lasts 5–10 years. LoRaWAN adds the MAC layer and network server. Perfect for smart agriculture, asset tracking, water meters.

LPWAN: Low Power Wide Area Network

Zigbee & Thread / Matter

Zigbee: 802.15.4 mesh networking, 250kbps, self-healing mesh up to 65,000 nodes. Smart home standard (Philips Hue, IKEA). Thread: IP-based 802.15.4 mesh (IPv6). Matter: unified smart home standard built on Thread/WiFi, backed by Apple/Google/Amazon — the future of smart home.

Cellular IoT (NB-IoT / LTE-M)

NB-IoT (Narrowband IoT): up to 10km, ultra-low power, penetrates deep into buildings. LTE-M: higher data rate, supports voice, mobile devices. Both use existing cellular infrastructure. Ideal for smart meters, fleet tracking, and any scenario needing cellular coverage without WiFi infrastructure.

Time-Series Databases

IoT data is inherently time-stamped. Regular SQL databases struggle at millions of sensor readings. TSDBs optimise for time-ordered writes and range queries: InfluxDB (most popular), TimescaleDB (PostgreSQL extension), OpenTSDB (Hadoop-backed), Prometheus (monitoring). Can store billions of data points efficiently.

Device Twin / Shadow

AWS IoT Device Shadow and Azure Device Twin maintain a JSON document representing device state — even when the device is offline. The shadow stores reported state (from device) and desired state (from app). When device reconnects, it syncs with shadow. Eliminates need for always-on connections.

Chapter 8 — Security

IoT Security — The Critical Challenge

IoT security is one of the most serious challenges of our time. Billions of resource-constrained devices are deployed in physical spaces, many with weak or no security, creating an unprecedented attack surface. The Mirai botnet (2016) hijacked 600,000 IoT devices to launch a record DDoS attack.

IoT ATTACK SURFACE & THREAT MODEL

The Mirai Botnet (2016)

Malware scanned the internet for IoT devices using default credentials (admin/admin). Infected 600,000+ cameras, DVRs, routers. Launched 1.2 Tbps DDoS attack — took down GitHub, Netflix, Twitter. Wake-up call for IoT security. Many countries now mandate unique default passwords by law.

Security Best Practices

• Unique device credentials per unit
• TLS 1.2+ for all communication
• Signed OTA firmware updates
• Secure boot (verify firmware)
• Hardware Security Module (HSM/TPM)
• Network segmentation (IoT VLAN)
• Minimal open ports
• Regular vulnerability scanning
• OWASP IoT Top 10 checklist

PKI & Certificate Auth

X.509 certificates with mutual TLS (mTLS) is the gold standard. Each device has a unique certificate signed by a Certificate Authority. Both device and server authenticate each other. AWS IoT Core and Azure IoT Hub both support mTLS per device. Certificates can be revoked remotely if a device is compromised.

Node-RED — Visual IoT Programming

Node-RED is a flow-based visual programming tool from IBM, built on Node.js. Drag and drop nodes: MQTT input → function node → InfluxDB output → email alert. No code needed for many IoT workflows. Runs on Raspberry Pi, Docker, or cloud. 4,000+ community nodes available.

TinyML — AI at the Edge

TensorFlow Lite Micro enables ML inference on microcontrollers with <256KB RAM. Train a model in TensorFlow, convert to TFLite, quantise to int8, deploy to Arduino Nano 33 BLE Sense or ESP32. Use cases: wake word detection, gesture recognition, vibration analysis, anomaly detection — all without cloud!
